针对开发人员解决方案设置文件

注释  此主题介绍选中"使用高级工具"常规首选项时可以使用的功能。请参阅使用高级工具。

FileMaker Pro Advanced 提供了"开发工具"，可让您：

•将数据库文件绑定至独立的 Runtime 数据库解决方案，该方案不需要 FileMaker Pro Advanced 即可在计算机上使用。

•加密数据库文件以防止在这些文件存储在磁盘上时对其进行未授权的访问。

•重命名一组数据库文件，并自动更新相关文件和脚本的内部链接。

•在 Kiosk 模式下显示数据库文件

•删除所有帐户的管理员访问权限，并防止用户修改数据库的大部分设计或结构元素

•将 FileMaker Pro 12 文件扩展名 (.fmp12) 添加到一组非 Runtime 文件

•创建处理错误的错误日志

注释  FileMaker Pro Advanced 允许您根据需要在数据库文件中包含任意数量的数据库表。使用多个文件有几个主要原因，此功能便抵消了其中一个原因。不过，其他元素（如脚本和访问权限）会存储在文件级别；因此，部分复杂的解决方案将受益于使用多个文件。

要针对开发人员解决方案设置文件：

1. 关闭所有您要自定义的数据库文件。

2. 选择"工具"菜单 >"开发工具"。

3. 如果您曾对同一数据库解决方案使用了"开发工具"并保存设置，则单击"加载设置"。

随即打开"选择文件"对话框，您可以在其中浏览以寻找设置文件。

4. 单击"添加"，找到您要自定义的文件。

5. 在添加要自定义的所有文件之后，执行下列一个或多个操作：

 

目的

请执行此操作

选择 Runtime 解决方案的主文件

双击列表中的文件。

重命名文件

在列表中选择文件，并在"重命名文件"框中键入新名称，然后单击"更改"。这还会更新相关文件和脚本的任何内部链接。

注释  不要键入文件扩展名。有关将文件扩展名添加到 Runtime 文件的更多信息，请参阅命名 Runtime 解决方案。

删除文件

选择列表中的文件，然后单击"删除"。

6. 要显示"项目文件夹"，请单击"指定"以选择解决方案的位置。

7. 在"选择文件夹"对话框中，选择或创建文件夹，并单击"确定"(Windows) 或"选择"(macOS)。

8. 如果不想让新文件改写以前版本的文件，请清除"改写项目文件夹内匹配的文件"复选框。

重要事项  如果选择"改写项目文件夹内匹配的文件"，"开发工具"会改写文件列表中同名的文件。

9. 要显示"解决方案选项"，请单击"指定"并选择选项。例如，可以自定义数据库文件或将文件绑定至 Runtime 解决方案。请参阅指定开发人员解决方案选项。

10. 为了能够快速重复这个过程，请单击"保存设置"，并为设置文件选择文件夹和位置。请参阅保存解决方案设置。

11. 单击"创建"。

"开发工具"会将所有选定数据库文件（包括该工具所做的修改）拷贝到"项目文件夹"。如果改写目的地文件的默认选项仍为打开状态，则指定的"项目文件夹"将用来存储"开发工具"拷贝的文件。如果已清除改写文件的默认选项，则在指定的"项目文件夹"为空时会使用该文件夹；否则会根据"项目文件夹"的名称创建新文件夹。
