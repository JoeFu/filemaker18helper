关于创建 Runtime 解决方案

注释  此主题介绍选中"使用高级工具"常规首选项时可以使用的功能。请参阅使用高级工具。

可以将 FileMaker 数据库文件绑定至 Runtime 解决方案，这些解决方案不需要 FileMaker Pro Advanced 即可在计算机上使用。也可以在 Kiosk 模式下创建 Runtime 解决方案。

在开始构建数据库解决方案之前，需要决定用户与解决方案交互的方式。数据库解决方案可能有下列任一组件：

•主数据库文件，用于连接所有辅助文件

•脚本和按钮，用于打开辅助文件、返回到主文件、启动时显示引导画面布局或退出 Runtime 应用程序

•适用于跨平台解决方案的共同元素和一致的外观

•工具提示和自定义菜单

•"关于"布局，用于介绍您的解决方案

•自定义"帮助"系统，提供解决方案的使用提示

•多个权限集，可以指定布局、菜单、特定表、记录、字段等的访问级别

•受密码保护的帐户，分配给用于决定帐户用户的访问级别的权限集

高级工具已从 Runtime 应用程序中去除。Runtime 应用程序中没有任何"文件"菜单 >"管理"子菜单上的命令。Runtime 应用程序无法在网络上共享，并且不包括"将记录保存/发送为"PDF 文件的功能。Runtime 应用程序中不支持关系图中的 ODBC 导入、"执行 SQL 语句"脚本步骤和使用 ODBC 数据源。请参阅 Runtime 应用程序和 FileMaker Pro Advanced 之间的差异。

但是，Runtime 数据库可以在 FileMaker Pro Advanced 中打开。这些应用程序的全部功能将被启用，除非已删除完全访问权限。请参阅删除数据库的管理员访问权限。

您可能需要将数据库文件绑定数次，才能准备将文件传递给用户。当完成开发并绑定最终版本以备分发时，应该全面测试 Runtime 解决方案以确保能够按预期运行。请参阅准备 Runtime 解决方案的文件。

注释 

•如果您创建的解决方案有针对 Windows 和 macOS 两种版本，请在其各自的平台上测试解决方案的不同版本。

•如果已使用多个文件，而不是在单个文件中使用多个表，则在绑定至 Runtime 解决方案之前，解决方案的所有文件应位于同一个文件夹中。如果在一个文件夹中保存所有文件的做法不可行，请确保加入每个文件的数据源参考（只是文件名）。

•务必指定与数据库解决方案相关的每个文件，以便在您修改文件名时会更新所有数据源。

•如果已使用多个数据库文件，请决定哪个文件将是用户最先打开的主文件。主文件存储自定义设置。使用此文件来存储其他辅助文件、"关于"布局、自定义"帮助"布局或文件，以及退出应用程序的导航按钮或脚本。请参阅启动 Runtime 解决方案。

•创建脚本和按钮，以便用户从解决方案中的主文件浏览到辅助文件和布局。

•如果将数据库作为需要特定插件版本的 Runtime 解决方案分发，请将插件放置在绑定解决方案时创建的 Runtime 文件夹中。

•图表在 Runtime 解决方案中不受支持。

•创建有关数据库解决方案的文档。请参阅记录开发人员解决方案。

•尽管"开发工具"使用的是文件副本而不是原始文件，仍建议您在开始之前先创建原始文件的备份副本。
