在网络上共享文件

如果计算机已连接到网络，则您和其他 FileMaker Pro Advanced 或 FileMaker Go 用户可以同时使用同一数据库文件。

FileMaker 主机

要共享某个文件以供其他用户处理，请使用下列 FileMaker 主机之一：

•FileMaker Pro Advanced：当您打开本地文件并启用 FileMaker Pro Advanced 网络共享之后，FileMaker Pro Advanced 可以充当主机并通过局域网同时与最多五个其他用户共享文件。FileMaker Go 客户端也会占用此限值。使用 FileMaker Pro Advanced 作为主机也称为点对点共享。请参阅以主机角色打开文件。

•FileMaker Server 和 FileMaker Cloud for AWS：在您上传文件并使 FileMaker Cloud for AWS 打开该文件后，服务器成为主机。这些主机支持更多用户和 Web 技术，如 FileMaker WebDirect 和 FileMaker Data API。参阅产品文档中心中的 FileMaker Server 帮助和 FileMaker Cloud for AWS 文档。

要在 Web 或 Intranet 上发布文件，请参阅在 Web 上发布数据库。

重要事项  请勿将数据库文件置于共享文件夹中而让多个设备同时将其打开，这样会损坏文件。相反，您必须使用 FileMaker 主机打开文件，这样主机才能与其他用户共享该文件。

在共享文件中工作

在主机之后打开共享文件的用户成为客户端角色。

注释  FileMaker 主机软件仅允许连接最近版本的 FileMaker 客户端软件。有关主机软件的兼容版本，请参阅 FileMaker Pro Advanced 技术规范。

一旦打开共享文件，主机和客户端即可访问同一信息，并且在每个用户窗口中都显示所有文件的更改。

•任何用户进行的数据修改都保存在共享文件中。布局和脚本的任何更改也保存在共享文件中。

•指定的排序顺序、查找请求、导入和导出字段顺序以及打印设置对于每个用户都是特定的。

尽管多个用户可以同时在某一共享文件中进行更改，但是对允许同时更改的类型是有限制的。

 

更改为

限制

字段和记录中的数据

两个人不能同时编辑同一记录。

布局和布局元素

两个人不能同时修改同一布局。

值列表

两个人不能同时修改或定义值列表。

在浏览模式或查找模式下，当某个人使用值列表时，另一个人可对其编辑。

脚本

两个人不能同时修改同一脚本。

当某个人使用脚本时，另一个人可以对其编辑。

表、字段、关系（数据库架构）

两个人不能同时修改任何此类元素。

数据源参考

两个人不能同时修改或定义数据源。

当某个人使用数据源时，另一个人可以对其编辑。

帐户访问权限和权限集

两个人不能同时修改或定义帐户访问权限和权限集。

当某个人使用帐户访问权限条目或权限集时，另一个人可对其进行创建或修改。

注释 

•FileMaker Pro Advanced 数据库的共享（或托管）与任何操作系统文件共享无关。在 Windows 或 macOS 中设置的访问控件与在 FileMaker Pro Advanced 中设置的访问权限无关，反之亦然。

•访问权限可能限制用户更改文件中某些元素的能力。请参阅管理安全性。
